Tổng quan

QuraEx Backend là gì?

Backend microservices cho nền tảng QuraEx — quản lý user story, test artifact, sinh test bằng AI, thực thi và tích hợp.

Nền tảng

.NET 10 · Vertical Slice + CQRS/MediatR

Giao tiếp

Event qua MassTransit / RabbitMQ

Cổng vào

YARP Gateway — định tuyến + xác thực JWT

Dữ liệu & dev

PostgreSQL (1 DB/service), Redis, điều phối local bằng .NET Aspire
Kiến trúc

8 service theo nghiệp vụ

Client → Gateway (YARP) → từng service độc lập, mỗi service có DB riêng và phát/nhận event qua RabbitMQ.

Client / FE→ YARP Gateway→ Identity Workspace Authoring Test-Artifact
AI-Generation Execution Integration Notification ⇄RabbitMQ (events)

Mỗi service = 4 lớp: Api · Domain · Infrastructure · Contracts (event công khai) + bộ Tests.

Quyết định kiến trúc

3 lựa chọn cần hiểu (hay bị hỏi)

① Monorepo — KHÔNG tách mỗi service 1 repo

Microservice = độc lập lúc deploy/runtime, không phải số lượng repo. Tách repo lúc này = gánh versioning, CI ×8, mất Aspire F5 — chưa có lợi ích khi đội nhỏ.

② GitHub Organization QuraEx

Chứa repo BE (và FE sắp dựng). Cho namespace package, teams/RBAC, secret dùng chung — độc lập với chuyện tách repo.

③ Database-per-service + Outbox pattern

Mỗi service 1 DB riêng, không share. Nhất quán dữ liệu qua event + transactional outbox, không cần distributed transaction.

Cấu trúc repo

Đường đi trong codebase

  • gateway/ — YARP reverse proxy + JWT
  • services/<name>/ — từng service (Api/Domain/Infrastructure/Contracts/Tests)
  • building-blocks/ — Result, BaseEntity, MediatR behaviors, Outbox, EF conventions
  • aspire/ — AppHost (điều phối) + ServiceDefaults (OTel, health)
  • deploy/ — compose prod, runbook, droplet setup
  • docs/ — kiến trúc, SRS, DB conventions, DBML, TASKS
  • docker-compose*.yml — infra + app services
  • QuraEx.slnx · Directory.Packages.props — solution + quản lý package tập trung
Bắt đầu

Chạy local trong 4 bước

1 · Clone

git clone git@github.com:QuraEx/QuraEx-BE.git && cd QuraEx-BE/quraexv2

2 · Restore

dotnet tool restore && dotnet restore QuraEx.slnx

3 · Chạy (chọn 1)

Aspire (dev hằng ngày, hot reload, debug):
dotnet run --project aspire/QuraEx.AppHost/QuraEx.AppHost.csproj

Docker full stack (không cần SDK): make up

4 · Verify

curl <gateway>/api/authoring/health → 200 OK
